# 绿联 NAS Docker 部署说明（纯 UI 操作 · 镜像拉取）

适配机型：**DXP4800 Plus** (Intel 8505 x86_64)
其他绿联 x86 NAS 同样适用。

## 原理

所有服务端程序已打包为 Docker Hub 镜像，部署时自动拉取，**无需编译、无需构建、无需命令行**。

---

## 一、准备文件

### 1. 上传 nas/ 目录到 NAS

将整个 `nas/` 文件夹上传到绿联 NAS，如：

```
/volume1/docker/wow/nas/
```

目录结构：
```
nas/
├── docker-compose.yml          # Docker 编排文件
├── .env.dist                   # 环境变量模板
├── 部署说明.md                  # 本文档
└── scripts/
    ├── realm-init.sh           # Realm 地址初始化
    └── create-default-account.py # 管理员账户创建
```

### 2. 放置客户端地图数据

在 `nas/data/` 下放入地图文件：

```
nas/data/
├── maps/
├── dbc/
├── vmaps/
├── mmaps/
└── cameras/
```

> 约 2-3GB，用 AzerothCore 提取工具从 3.3.5a 客户端导出。

### 3. 创建 .env 文件

在绿联文件管理器中：
1. 进入 `nas/` 目录
2. 复制 `.env.dist` → 重命名为 `.env`
3. 编辑 `.env`，**至少修改 REALM_IP**

```
REALM_IP=192.168.1.100   # 改成 NAS 的局域网 IP
```

---

## 二、绿联 Docker UI 部署

### 1. 打开 Docker 应用

绿联 NAS 桌面 → **Docker**

### 2. 新建项目

点击 **「项目」** → **「新建项目」**

### 3. 填写信息

| 字段 | 内容 |
|------|------|
| 项目名称 | `wow` |
| 项目文件夹路径 | `/volume1/docker/wow/nas/` |

### 4. 粘贴 Compose 配置

将 `nas/docker-compose.yml` 的全部内容粘贴到编辑器，或点击「从文件导入」。

### 5. 部署

点击 **「部署」** 按钮：
- 自动拉取 `asm0x1/wow-worldserver`、`asm0x1/wow-authserver`、`asm0x1/wow-registration` 三个镜像
- 自动拉取 `mariadb:10.11`、`python:3.11-alpine` 公共镜像
- 首次约 5-10 分钟（取决于网速），之后自动启动全部服务

## 三、确认运行状态

部署完成后，在 Docker → 容器页面查看：

| 容器 | 预期状态 |
|------|----------|
| acore-database | ✓ 运行中 |
| acore-worldserver | ✓ 运行中 |
| acore-authserver | ✓ 运行中 |
| acore-registration | ✓ 运行中 |
| acore-realm-init | 已退出(0) ← 正常 |
| acore-account-init | 已退出(0) ← 正常 |

### 查看日志

点击 `acore-worldserver` → 日志，确认看到：

```
>> AI Playerbots initialized
>> 53 random bot accounts with 530 characters available
```

## 四、客户端登录

1. 魔兽世界 3.3.5a 客户端 → `Data/zhCN/realmlist.wtf` 改为：
   ```
   set realmlist NAS的局域网IP
   ```
2. 运行 `Wow.exe` 登录
3. 内置账号：`asm0x1` / `123456`
4. Web 注册：浏览器访问 `http://NAS_IP:8765`

## 五、管理操作（全部在 Docker UI）

| 操作 | 位置 |
|------|------|
| 启动/停止全部 | 项目 → wow → 操作 |
| 重启某个服务 | 容器 → 点击容器 → 重启 |
| 查看日志 | 容器 → 点击容器 → 日志 |
| GM 控制台 | 容器 → acore-worldserver → 终端 |

### GM 命令（在 acore-worldserver 终端中执行）

```
account create 用户名 密码          # 创建账号
account set gmlevel 用户名 3 -1     # 设为管理员 (GM 3)
account onlinelist                  # 查看在线玩家（含假人）
```

---

## 六、调整假人数量

1. 文件管理器 → 编辑 `nas/.env`
2. 修改：
   ```
   BOT_MIN_COUNT=10
   BOT_MAX_COUNT=30
   ```
3. Docker → 容器 → 重启 `acore-worldserver`

| NAS 内存 | 建议假人数 |
|----------|------------|
| 4GB | 5-10 |
| 8GB | 15-30 |
| 16GB+ | 30-100 |

---

## 七、更新镜像

当有新版本镜像推送后：

1. Docker → 容器 → 删除 `acore-worldserver`、`acore-authserver`、`acore-registration`
2. Docker → 镜像 → 删除旧版镜像
3. Docker → 项目 → wow → 重新部署

---

## 八、完全重置

清除所有数据：

1. Docker → 项目 → wow → 操作 → **删除项目**
2. 勾选「同时删除卷」
3. 重新创建项目部署

---

## 九、防火墙

如 NAS 启用了防火墙，放行：

| 端口 | 说明 |
|------|------|
| 3724 (TCP) | 游戏登录 |
| 8085 (TCP) | 游戏世界 |
| 8765 (TCP) | Web 注册 |

---

## 十、启用 phpMyAdmin（可选）

编辑 `nas/docker-compose.yml`，取消 `phpmyadmin` 段的注释，重新部署。

访问 `http://NAS_IP:8081`，用户名 `root`，密码见 `.env` 中 `DOCKER_DB_ROOT_PASSWORD`。
