# 绿联 NAS Docker 部署说明（纯 UI 操作 · 镜像拉取）

适配机型：**DXP4800 Plus** (Intel 8505 x86_64)，其他绿联 x86 NAS 同样适用。

---

## 部署步骤（3 步）

### 1. 准备文件

在 NAS 文件管理器中创建项目目录，如 `/volume1/docker/wow/`，放入：

```
docker/wow/
└── data/                          # 客户端地图数据（约 2-3GB）
    ├── maps/
    ├── dbc/
    ├── vmaps/
    ├── mmaps/
    └── cameras/
```

> 地图数据用 AzerothCore 提取工具从 3.3.5a 客户端导出。[提取教程](https://www.azerothcore.org/wiki/client-setup)

### 2. 修改配置（只改 2 个值）

打开 `nas/docker-compose.yml`，文件**最顶部**：

```yaml
x-deploy-config:
  REALM_IP:      192.168.1.100   # ← 改成 NAS 的局域网 IP
  DB_ROOT_PASS:  wow@asm0x1       # ← 按需修改数据库密码
```

下面所有服务自动引用这两个值，**不需要在文件里搜索替换**。

### 3. 部署

1. 绿联 NAS 桌面 → **Docker** → **项目** → **新建项目**
2. 项目名称：`wow`
3. 项目文件夹路径：选择步骤 1 创建的目录（如 `/volume1/docker/wow/`）
4. 将修改后的 `nas/docker-compose.yml` **全部内容粘贴**到编辑器
5. 点击 **「部署」**

首次部署自动拉取镜像，约 5-10 分钟。

## 确认运行状态

Docker → 容器：

部署后共 4 个容器，全部运行中，无已退出的一次性容器：

| 容器 | 预期状态 |
|------|----------|
| acore-database | ✓ 运行中 |
| acore-worldserver | ✓ 运行中 |
| acore-authserver | ✓ 运行中（启动时自动完成 Realm 初始化和账户创建） |
| acore-registration | ✓ 运行中 |

点击 `acore-authserver` → 日志，确认：

```
>> [entrypoint] 更新 Realm 地址: 192.168.1.100:8085
>> [entrypoint] Realm 地址已设置完成
>> 账户初始化完成!
>> Added realm "群晖魔兽世界" at 192.168.1.100:8085
```

如果 IP 不对，回到步骤 2 改 `REALM_IP`，重新部署。

   ```
   set realmlist NAS的局域网IP
   ```

3. 内置账号：`asm0x1` / `123456`
4. Web 注册：`http://NAS_IP:8765`

## GM 命令

Docker → 点击 `acore-worldserver` → **终端** 执行：

```
account create 用户名 密码            # 创建账号
account set gmlevel 用户名 3 -1       # 设为管理员
account onlinelist                    # 查看在线玩家（含假人）
```

## 调整假人数量

编辑 YAML 中 `x-bot-config` 部分：

```yaml
x-bot-config:
  BOT_MIN_COUNT:     10    # 最少在线假人
  BOT_MAX_COUNT:     30    # 最多在线假人
```

| NAS 内存 | 建议假人数 |
|----------|------------|
| 4GB | 5-10 |
| 8GB | 15-30 |
| 16GB+ | 30-100 |

修改后重新部署生效。

## 配置说明

`nas/docker-compose.yml` 顶部使用 YAML 锚点定义所有配置：

```yaml
x-deploy-config:
  REALM_IP:      &REALM_IP      192.168.1.100  # ← 只改这里
  DB_ROOT_PASS:  &DB_ROOT_PASS  wow@asm0x1      # ← 只改这里

x-port-config:
  REALM_PORT:    &REALM_PORT    8085            # 通常无需修改
  ...

x-bot-config:
  BOT_MIN_COUNT: &BOT_MIN_COUNT 15             # 假人数量
  ...
```

`&name` 定义锚点，`*name` 引用锚点。改顶部值，下面所有 `*REALM_IP`、`*DB_ROOT_PASS` 自动生效。

## 更新镜像

1. Docker → 容器 → 删除 `acore-worldserver`、`acore-authserver`、`acore-registration`
2. Docker → 镜像 → 删除旧版 `asm0x1/wow-*` 镜像
3. Docker → 项目 → wow → 重新部署

---

## 完全重置

1. Docker → 项目 → wow → **删除项目**
2. **勾选「同时删除卷」**
3. 重新创建项目部署
