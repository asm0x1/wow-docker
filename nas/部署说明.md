# 绿联 NAS Docker 部署说明（纯 UI 操作）

适配机型：**DXP4800 Plus** (Intel 8505 x86_64)
其他绿联 x86 NAS 同样适用。

## 架构说明

- **CPU**: Intel 8505 是 x86_64 架构，Docker 原生运行，无需模拟
- **内存**: 建议 8GB+，worldserver + 假人系统约占用 2-4GB
- **磁盘**: 客户端地图数据约 2-3GB

---

## 一、准备文件

### 1. 在绿联 NAS 文件管理器中创建目录

```
/volume1/docker/wow-docker/
```

将项目所有文件上传/复制到此目录。确保包含：
- `Dockerfile.worldserver`、`Dockerfile.authserver`、`Dockerfile.registration`
- `docker-compose.yml`（或在 Nas Docker UI 中手动粘贴）
- `bin/`、`lib/`、`database/`、`scripts/`、`conf/`、`sql/`、`WoWRegistration/`

### 2. 放入客户端地图数据

在 `/volume1/docker/wow-docker/data/` 下放入：

```
data/
├── maps/
├── dbc/
├── vmaps/
├── mmaps/
└── cameras/
```

这些文件约 2-3GB，从魔兽 3.3.5a 客户端用提取工具导出。

### 3. 创建 .env 环境变量文件

在文件管理器中，复制 `nas/.env.dist` 到项目根目录，重命名为 `.env`，然后修改：

**必改项：**
```
REALM_IP=你的NAS局域网IP
```

> NAS IP 查看方式：绿联客户端 → 设备信息 → IP 地址，通常是 `192.168.x.x`

---

## 二、在绿联 Docker 管理界面部署

### 步骤 1：打开 Docker 应用

进入绿联 NAS 桌面 → 打开 **Docker** 应用

### 步骤 2：新建项目

1. 点击左侧 **「项目」**
2. 点击右上角 **「新建项目」** 按钮

### 步骤 3：填写项目信息

| 字段 | 填写内容 |
|------|----------|
| **项目名称** | `wow-docker` |
| **项目文件夹路径** | `/volume1/docker/wow-docker/` |

### 步骤 4：粘贴 Docker Compose 配置

在 Compose 配置编辑器中，**全选删除默认内容**，然后粘贴 `nas/docker-compose.yml` 的完整内容：

```yaml
# 复制 nas/docker-compose.yml 全部内容粘贴到这里
```

> 也可直接点击「从文件导入」选择 `nas/docker-compose.yml`

### 步骤 5：部署

1. 点击 **「部署」** 按钮
2. 等待镜像拉取和构建完成（首次约 5-10 分钟）
3. 看到所有服务状态为 ✓ 运行中即成功

---

## 三、验证运行状态

### 查看容器状态

在 Docker → 容器页面，确认以下容器都在运行：

| 容器名 | 说明 |
|--------|------|
| acore-database | 数据库 (MariaDB) |
| acore-worldserver | 世界服务器 |
| acore-authserver | 认证服务器 |
| acore-registration | Web 注册页面 |
| acore-realm-init | 已退出(0) — 一次性初始化，正常 |
| acore-account-init | 已退出(0) — 一次性初始化，正常 |

### 查看 worldserver 日志

在 Docker → 容器页面，点击 `acore-worldserver` → 日志，确认看到：

```
>> AI Playerbots initialized
```

### 确认假人已上线

在 `acore-worldserver` 日志中搜索 `playerbot`：

```
53 random bot accounts with 530 characters available
AI Playerbots initialized
```

## 四、客户端登录

1. 魔兽世界 3.3.5a 客户端目录下，将 `Data/zhCN/realmlist.wtf` 改为：
   ```
   set realmlist NAS的局域网IP
   ```
2. 运行 `Wow.exe` 登录
3. 内置账号：`asm0x1` / `123456`
4. 或浏览器访问 `http://NAS_IP:8765` 注册新账号

---

## 五、服务管理（全部在 Docker UI 完成）

| 操作 | 位置 |
|------|------|
| 启动/停止 | 项目页面 → 操作 → 启动/停止 |
| 重启某个服务 | 容器页面 → 点击容器 → 重启 |
| 查看日志 | 容器页面 → 点击容器 → 日志 |
| 进入控制台 | 容器页面 → 点击 `acore-worldserver` → 终端 |

### 在控制台中执行 GM 命令

进入 `acore-worldserver` 容器终端后：
```
account create 用户名 密码          # 创建账号
account set gmlevel 用户名 3 -1     # 设为管理员
account onlinelist                  # 查看在线玩家
```

## 六、调整假人数量

1. 在文件管理器中编辑 `/volume1/docker/wow-docker/.env`
2. 修改以下值：
   ```
   BOT_MIN_COUNT=10    # 最少在线假人
   BOT_MAX_COUNT=30    # 最多在线假人
   ```
3. 在 Docker → 容器页面，重启 `acore-worldserver`

| NAS 内存 | 建议假人数 |
|----------|------------|
| 4GB | 5-10 |
| 8GB | 15-30 |
| 16GB+ | 30-100 |

---

## 七、完全重置

如需清空所有数据重新开始：

1. Docker 应用 → 项目 → `wow-docker` → 操作 → **删除项目**
2. 勾选「同时删除卷 (volumes)」确认
3. 按步骤二重新创建项目部署

---

## 八、防火墙设置

如果 NAS 启用了防火墙，需要在绿联控制面板 → 防火墙中放行以下端口：

| 端口 | 协议 | 说明 |
|------|------|------|
| 3724 | TCP | 游戏登录 |
| 8085 | TCP | 游戏世界 |
| 8765 | TCP | Web 注册页面 |
| 63306 | TCP | 数据库 (可选) |

---

## 九、启用 phpMyAdmin（可选）

如果需要在浏览器管理数据库：

1. 编辑 compose 配置，取消 `phpmyadmin` 段的注释
2. 重新部署
3. 访问 `http://NAS_IP:8081`
4. 用户名 `root`，密码见 `.env` 中的 `DOCKER_DB_ROOT_PASSWORD`
